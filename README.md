# SDD Framework — Spec-Driven QA Automation

A complete Spec-Driven Development (SDD) framework for QA Automation Engineers.
Converts Jira acceptance criteria into Excel test cases, Gherkin .feature files,
and executable Selenium + REST-assured + Axe automation.

## Quick Start

### Step 1 — Configure
Edit `resources/config/config.properties`:
```properties
app.base.url=https://your-app.com
api.base.url=https://api.your-app.com
browser=chrome
```

### Step 2 — Generate Test Cases from AC
Open `src/main/java/com/sdd/MainRunner.java`:
1. Set your `ticketId`
2. Paste your Jira acceptance criteria
3. Run as Java main application

**Output:**
- `output/test-cases-PROJ-101.xlsx` — Manual test cases (4 sheets: Summary, UI, API, Accessibility)
- `output/traceability-PROJ-101.html` — HTML traceability matrix
- `src/test/resources/features/` — Cucumber .feature files (ready to run)

### Step 3 — Update Page Locators
Edit `src/test/java/com/sdd/pages/LoginPage.java`:
```java
private final By emailField    = By.id("YOUR_EMAIL_ID");
private final By passwordField = By.id("YOUR_PASSWORD_ID");
private final By loginButton   = By.id("YOUR_LOGIN_BTN_ID");
```

### Step 4 — Run Automation
```bash
# Run all tests
mvn clean test

# Run only UI tests
mvn clean test -Dcucumber.filter.tags="@ui"

# Run only API tests
mvn clean test -Dcucumber.filter.tags="@api"

# Run only critical tests
mvn clean test -Dcucumber.filter.tags="@critical"
```

### Step 5 — Generate Reports
```bash
# Serenity living documentation
mvn serenity:aggregate
# Open: target/site/serenity/index.html

# Allure report
allure serve output/allure-results
```

## Adding a New Ticket

1. Open `MainRunner.java`
2. Change `ticketId = "PROJ-102"`
3. Paste new AC
4. Run → new Excel + .feature files auto-generated

## Requirements

- Java 11 or higher
- Maven 3.8+
- Chrome browser (or Firefox — set `browser=firefox` in config)
- Internet connection (WebDriverManager auto-downloads driver)
